Mini-Shell en C

Description du projet


Ce projet vise à implémenter un mini-shell en langage C qui simule certaines fonctionnalités d'un shell Unix classique. Il traite des commandes entrées par l'utilisateur, gère les arguments, les redirections et les pipelines.

Le projet est divisé en plusieurs exercices qui progressent en complexité, offrant un aperçu complet de la programmation système en C.


Exercice 1 : Commandes sans arguments


Implémentation d'un shell acceptant des commandes sans arguments. Le shell exécute ces commandes et attend leur terminaison avant de revenir à l'invite de commande.

Exécuter ls, pwd, whoami sans arguments.

Taper exit pour quitter le shell.


Exercice 2 : Traitement des arguments


Modification du shell pour accepter jusqu'à 16 arguments après la commande.

  • Taper echo Hello World et vérifier que les arguments sont bien passés à la commande.

Exercice 3 : Redirection de la sortie standard


Le shell peut maintenant rediriger la sortie d'une commande vers un fichier.

  • Exécuter echo Hello World > test.txt et vérifier le contenu de test.txt.

Exercice 4 : Tubes simples


Implémentation d'une version simplifiée du chaînage de commandes avec un tube.

  • Chaîner ls | wc -l pour compter les fichiers dans le répertoire courant.

Exercice 5 : Signaux


Le shell gère maintenant le signal SIGINT pour interrompre les programmes en cours d'exécution.

  • Lancer sleep 10 puis appuyer sur CTRL+C pour interrompre le sommeil.

Conclusion


Ce projet de mini-shell en C permet de comprendre les bases de la création d'un interpréteur de commandes, de la gestion des processus et des signaux dans un système d'exploitation. Il offre une bonne introduction à la programmation système en C.


Pour en savoir plus sur les fonctionnalités et le code source du Shell, le projet est consultable sur GitHub.
